Мини-гайд по Git

Основы, ветки, Pull Request, Stash, Cherry-pick, Reset и Revert

1. Что такое Git

Git — система контроля версий. Хранит историю проекта и позволяет:

  • Возвращаться к старым версиям кода
  • Работать в команде без потери данных
  • Создавать параллельные ветки разработки

2. Git и GitHub

Git — инструмент для работы с версиями на вашем компьютере.

GitHub — онлайн-платформа, где хранятся удалённые репозитории.

Конвенция по именованию коммитов

  • feat: добавление нового функционала — feat: add search bar
  • fix: исправление ошибок — fix: search bar validation
  • docs: изменения в документации — docs: add info on loading
  • chore: работы «по дому», изменения зависимостей — chore: add prettier
  • refactor: рефакторинг кода без добавления функционала
  • style: изменения стилей
  • build: исправления ошибок сборки
  • perf: улучшение производительности

Основные команды для работы с ветками

git checkout -b example-name     # создать и перейти в новую ветку
git checkout main                # перейти в ветку main
git branch name                  # создать ветку с указанным названием
git branch -a                    # посмотреть все ветки (* — текущая)
git branch -d dashboard          # удалить локальную ветку
git push origin --delete dashboard  # удалить удалённую ветку
git branch -m old_name new_name  # переименовать локальную ветку
git push -u origin advert        # создать удалённую ветку и залить туда код

Git Stash

Временное сохранение изменений:

git stash        # спрятать изменения
git stash pop    # достать изменения

Git Fetch

Обновить информацию об удалённом репозитории:

git fetch

Как жить в этом мире? (Процесс работы)

  1. Создать ветку: git checkout -b name_of_branch
  2. Написать код, выполнить: 
    git add .
    git commit -m 'your message'
  3. Подтянуть актуальное состояние: 
    git pull origin main
    или 
    git pull origin dev
  4. Разрешить конфликты
  5. Отправить ветку: 
    git push -u origin name_of_branch
  6. Создать Pull Request
  7. Team Lead мержит или отклоняет PR

Первое правило гитовского клуба

Пушить только в одноимённую ветку. Между разными ветками — только через Pull Request.

Cherry-pick

Выборочный перенос коммита:

git cherry-pick <хэш коммита>
# При конфликтах:
# - исправляем конфликт в файлах
git add .
git cherry-pick --continue

Git Revert

git revert — команда, которая отменяет изменения конкретного коммита, создавая новый коммит, который «откатывает» его. В отличие от git reset, история изменений сохраняется.

Когда использовать

  • Когда коммит уже отправлен в удалённый репозиторий
  • Когда нужно безопасно отменить изменения, не ломая историю

Синтаксис

git revert <хэш_коммита>

Пример


# Смотрим историю
git log

# Откатываем коммит по хэшу
git revert abc123

Разрешение конфликтов при revert


git add .
git revert --continue

Отличие от git reset

git revert
Создаёт новый коммит, который отменяет изменения. История сохраняется. Безопасно для команды.
git reset
Перематывает историю назад, удаляя коммиты. Может быть опасно, если уже пушили в общий репозиторий.
